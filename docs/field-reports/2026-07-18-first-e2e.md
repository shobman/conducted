## Executive summary

The full cycle worked: genesis → interview → VISION/ROADMAP showcase PR → brief → countersign →
fleet → evaluator → showcase PR → owner merge → wake → rulings → next brief. The product outcome
was real (a proven pipeline plus a machine-verified 2-player divergent-perception playtest
harness). The cost pattern was not: the node landed at ~2x its predicted band, and roughly
**20–25% of node spend bought hygiene, not product** — an evaluation loop over self-authored
artifacts that does not terminate by design. Separately, the seed shipped three self-contradictions
that made the project's very first docs-only PR fail CI three times. Every item below is either
already fixed in the field (backport candidates, code exists and is gate-tested in the game project)
or is a skill/harness design recommendation.

## Part 1 — the seed-bug tax (three CI failures on the FIRST PR any project opens)

The genesis showcase is a docs-only PR. Ours failed gates three times, none about the content:

**1a. gitleaks 403 — missing workflow permissions.**
`seed/.github/workflows/gates.yml` has no `permissions:` block; gitleaks-action calls the PR
commits API and gets `Resource not accessible by integration`. Every new repo whose default token
policy is restrictive hits this on PR #1.
*Field fix (proven):* add `permissions: { contents: read, pull-requests: read }`.

**1b. The `node:` trailer grep eats prose.**
`gates.yml` derives the node from `grep -oP -m1 '(?<=node: ).*'` over commit messages. A genesis
commit carrying the reasonable trailer `node: none (docs-only genesis showcase)` was parsed as a
node named "none (docs-only genesis showcase)" and failed the brief-existence check. Recovery was
expensive out of proportion: history rewrite → force-push blocked by the owner's permission
classifier → branch superseded → PR #1 closed, PR #2 opened. Two design smells: (i) the
absence-of-trailer convention invites well-meaning explicit trailers that break it; (ii) worse,
omitting the trailer SKIPPED the territory gate entirely — the measured party chose whether it was
measured (independently flagged by an evaluator later, see 2b).
*Field fix (proven):* no trailer ⇒ the diff must be docs-only (`git diff ... -- ':!docs'` must be
empty), else fail loudly. See the game project's `gates.yml` @ 899f588.

**1c. The template contradicts the placeholder gate.**
`seed/templates/brief.md` ships `<FILES-AND-SURFACES-EXAMINED>`-style placeholders in the
countersign section (correctly — the conductor fills them later), but the gates' placeholder check
fails any `<[A-Z±][^>]*>` under `work/**`. A freshly-instantiated, fully-correct draft brief
cannot pass CI.
*Field fix (proven):* template countersign lines read `scouted: pending — conductor fills at
countersign` etc. Alternative: exempt the countersign section in the gate; the template fix is
simpler.

**1d. (Found by evaluator round 1, same class.)** Seed scripts committed without executable mode
bits let `[[ -x ./scripts/test.sh ]]`-style guards silently skip the entire suite — CI was green
while never running a test. The v0 gates.yml treated non-executable as "skip"; the node changed it
to a hard fail. Ensure seed scripts carry `100755` and the guard fails rather than skips.

*Why this section matters most per token spent:* these cost little compute but they are the NEW
USER'S FIRST HOUR with conducted. Three consecutive red X's on a docs-only PR reads as "this
system is broken", and the recovery (1b) burned real owner attention on git surgery.

## Part 2 — the territory gate hole (escalation 1)

Chronology, fully documented in `docs/findings/territory-gate-self-widening.md`:

1. Doctrine mandates immutable briefs + append-only deltas, but the seed's manifest parser only
   read globs from the original `## Territory manifest` section. **The delta mechanism doctrine
   mandates was unimplementable in the shipped tooling.** First contact (node needed `scripts/**`
   + one config file) hit the wall immediately.
2. Owner ruled the widening; conductor patched the parser to read `### Manifest delta` blocks —
   a node editing its own gate, disclosed, but forced by (1).
3. Evaluators then defeated the patched gate three different ways across two rounds: append a
   delta granting `**`; cite an unrelated same-day ruling (date-only matching); second delta block
   inheriting the first's authority. Each defeat triggered a hardening dispatch (~77K tokens) and
   fresh evaluation surface.
4. Node-level hardening cannot close it: both sides of the check (brief + DECISIONS) live in the
   node's own PR. The measured party writes the measurement.

**Field fix (proven, and it is a SIMPLIFICATION):** territory is law as of the BASE ref.
`manifest-check.sh` reads the brief — original manifest AND deltas — via `git show
"$BASE:$BRIEF"`, never the working tree. In-PR forgery becomes structurally impossible and ~150
lines of authority-checking heuristics get deleted. Mid-node widening flow: conductor escalates →
owner rules → Principal appends delta + ruling directly to main (minutes, no second PR) → the
node's PR passes naturally. Two companions landed with it: Principal law files
(VISION/ROADMAP/DECISIONS) are never node territory regardless of globs (v0 always allowed
`docs/**`, which is what made the DECISIONS half of the forgery available); nodes keep
`docs/findings/**` + `docs/ledger/**` + their own `work/<node>/`. Gate-tested red in both failure
modes before landing. Full script: the game project's `scripts/manifest-check.sh` @ 899f588; ruling
text in DECISIONS same date.

## Part 3 — evaluation non-convergence (escalation 2; the big token sink)

Four fresh-context evaluations, four FAILs, no convergence; the owner ruled a stop at round 4.
From `status.json`:

| round | probes | fails | class | what it found |
|---|---|---|---|---|
| 1 | 44 | 6 | engineering | missing ambient sound; CI never ran (mode bits, 1d); unrecorded ruling; parser glob leak |
| 2 | 35 | 4 | engineering | **no spawn point at all** — players spawned on the roof or fell out of the map; the node's own harness teleported past it and reported PASS |
| 3 | 47 | 3 | hygiene | 7/7 DoD passed; no showcase yet, stale ledger, gate defeat #1 |
| 4 | 56 | 10 | hygiene | 7/7 DoD passed again; gate defeats #2–3, status.json self-contradiction, showcase overstated a CI run, sensor gaps, zero-test guard |

Rounds 1–2 are the process **working** — the spawn bug is the single best catch of the entire
node and the conductor's own review missed it. Rounds 3–4 are the pathology:

- The conductor writes showcase/ledger/status.json about its own work, then an adversarial
  reader with unbounded scope judges the prose. **There is no test that can go red for prose
  before the evaluator looks** — no equivalent of "prove the gate fails".
- Each round's remediation creates the surface the next round faults (round 3's "write the
  showcase" fix became round 4's "showcase overstates a CI run" finding; round 3's gate hardening
  became round 4's two fresh defeats). Remediation that reliably generates findings does not
  terminate.
- The evaluator brief's "a zero-failure pass is suspicious" is correct and valuable — but
  combined with unbounded verdict scope it guarantees a diligent evaluator always returns
  something, and something is always available in documentation. Round 4 found MORE items (10)
  than round 3 (3) on a strictly better node.

Cost: each eval-fail cycle ≈ 400K tokens (opus evaluation + sonnet fix dispatch). Rounds 3–4 plus
their fix/hardening dispatches ≈ 350–470K of a 1.70M-token node — 20–25% of spend on hygiene
after the product was done. The node closed on an ad-hoc owner ruling made under budget pressure,
which is exactly what a stopping rule exists to avoid.

**Field ruling (recorded law, recommend baking into the skill):**
- Evaluator VERDICT is binary on the brief's DoD clauses. Everything else is recorded as advisory
  findings.
- A node may close when ALL DoD clauses pass in TWO CONSECUTIVE independent evaluations.
- Advisory findings become **inherited debt**: the Principal folds them into the next node's
  brief, where they block the NEXT dispatch, not this merge. (Six items carried on this project —
  the mechanism demonstrably works and findings don't evaporate.)
- Evaluator reports are committed artifacts under `work/<node>/evaluations/` (in v0 they lived
  only in subagent transcripts — itself a round-4 finding).
- KEEP unbounded probing. Scope the verdict, never the curiosity — the spawn bug came from an
  off-script probe.

### The owner's two prior-art analogies (his words, and they map cleanly)

- **aidos-auditor**: his audit tool deliberately allows 2 passes and tiers findings so
  "informational" ones aren't failures. The ruling above is that design plus a carry-forward
  mechanism so informational findings don't evaporate.
- **Devin PR-review bot, 3-strikes rule**: his team's Devin bot consistently finds new issues on
  unchanged code and sometimes REVERSES its own severity call between rounds ("not serious" in
  round 1, blocking in round 2); their countermeasure is closing round-3 re-findings on unchanged
  code as won't-fix. The conducted analog was round 4 re-finding categories round 3 implicitly
  passed. Recommendation is to fix it upstream of a strike counter: (a) DoD-scoped verdicts make
  hygiene re-litigation unable to block; (b) committed evaluator reports with **stable finding
  IDs** make "this exact item was already ruled on and its territory hasn't changed since" a
  mechanical check — sticky severity, re-escalation requires a diff touching the finding's files.
  That is the 3-strikes rule reduced to 1 strike + evidence.

## Part 4 — cost instrumentation

The ledger's per-dispatch table (15 dispatches, 1,703,503 subagent tokens, per-dispatch model +
wall clock) was reconstructed from the transcript at close; the orchestrating conductor session
was never measurable at all, and the ledger explicitly declines to invent a number. Hand-
reconstructed figures contradicting their own sources were a dominant round-3/4 defect class.

Recommendations:
- Skill: `docs/ledger/<node>.md` must exist from dispatch 1 and be APPENDED at each dispatch
  completion with that dispatch's `subagent_tokens` (recorded as law on this project; the
  vertical-slice brief makes it a DoD clause).
- Skill: generate status.json/ledger numbers mechanically from dispatch results wherever possible
  — less hand-written prose to be wrong (option D of the escalation finding).
- Harness ask (out of conducted's control, worth filing): expose a running token total for the
  orchestrating session. Instrumentation must be born, not restored.
- Estimation rule: **price one evaluation-failure cycle into every node estimate.** The $15–35
  band assumed zero fix rounds; actuals ran ~2x, driven almost entirely by two legitimate
  eval-fail cycles. An evaluator that never fails anything isn't working; the estimate should
  assume it works.

## Part 5 — what worked and must not be lost

- **Disjoint territory fleets:** four waves, zero file collisions, zero rework; the reserved
  best-of-N race for the riskiest wave (Studio harness) was never needed.
- **Fresh-context evaluation with unbounded probes:** caught the spawn bug, the never-run CI, and
  the gate defeats. All three would have shipped otherwise.
- **Escalate-before-acting:** both countersign escalations (evidence mechanism, manifest
  widening) produced owner rulings that held up; the one governance miss was recording a ruling
  late, not skipping it.
- **Honest ledgers:** "I decline to invent a number" and "roughly double the band" is exactly the
  culture the doctrine wants; protect it when adding mechanization.
- **Findings as product:** `docs/findings/toolchain.md` (Studio MCP no-go for playtests, the
  `RunScript`+`ExecuteMultiplayerTestAsync` mechanism, the three lying-gates gotchas) is arguably
  worth more than the node's code. The findings-gated edge did its job: the next brief was
  written absorbing all of it, plus the debt.
- **Owner-in-the-loop economics:** the appetite conversation (a low-hundreds cap → observed 2x
  overrun → an informed raise to a higher band with a priced recommendation) is the system behaving
  as designed.
