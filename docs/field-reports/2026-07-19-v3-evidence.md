## What each project proved

**the game project (vertical-slice)** — the v2 stopping rule is a stochastic ratchet, not a stopping rule.
"Two consecutive all-DoD-pass evaluations, any fail resets to zero" is not anchored to a tree:
round 2 passed 10/10 at `1ecef9d`, then 674 lines of post-pass fix/hardening commits changed the
clause-2/3 sensors, so the counter vouched for a tree that no longer existed. Round 3 then
legitimately failed clauses 2 and 7 on the CHANGED territory (structural sensor defeat — every
displacement threshold had a hop size beneath it), resetting the counter to zero at 3.39M tokens
with the v2 letter demanding two more FULL rounds (~190–210K each) on a node where eight clauses
had already passed twice on unchanged territory. Expected cost of two consecutive passes under
per-round pass probability p is (1+p)/p² full rounds — the tail the owner felt as "might be
impossible". Also from this node: the conductor authored two malformed sabotages whose red results
looked like proof ("the FAIL was real but meaningless — I only knew because I checked the cause
rather than the exit code"); a fix dispatch falsified the conductor's own suggested sensor signal
(Humanoid.MoveDirection reads 0,0 under MoveTo) — report-deviations working; and the round-1→2
history shows a same-node A/B for advisory severity: EV-1→EV-13 escalated on UNCHANGED code with
NEW evidence (3-of-3 reproduction) and was RIGHT — falsifying the v2 sticky-severity wording that
required a diff to re-escalate. Structural audit gaps found by interrogation: evaluator briefs
were never committed (the instruction to the measurer was unauditable), no per-round
evaluated-commit field, no EV-id resolution tracking, gates block never flipped.

**the queue app (consolidation)** — hand-off-ability fails silently without a forcing function. The closing
session claimed "nothing needs this conversation to be resumable"; an exit interview falsified it
by one category — the consolidation's judgment (triage method, four near-misses, the
stale-local-main trap) and the migration's input existed only in the session until asked. The
76→1 branch-GC ran with NO doctrine coverage (neither skill describes cleaning up after parallel
work); its evidence standard is now law: content-on-main proves (shared SHAs / gh merged-set /
grep hit), names never do; HELD is a hard stop; contradicted content is worse than useless — a
prior keep-decision is re-tested, not deferred to. Under squash-merge `git branch --merged` lies;
`gh pr list --state merged` is the authoritative source. Also fed up: "verify the invariant the
FAILURE would break, not one it happens to preserve" (five receipts), and the showcase-as-tour
shape. Economics baseline: consolidation is single-session, zero-fleet, git-plumbing-bound.

**the booking app (fabricated premise)** — a verification hole BETWEEN the countersign and the evaluator.
A fleet subagent's causal hypothesis (a service start-order race), born mid-execution, was
filed as an issue and fix-dispatched with zero runtime observations — the issue's own body
recorded that the environment's own telemetry had already disproved the race, and asserted it
anyway. Cost: ~35 min dev outage, ~63 min false trail, one dead 8-file branch, one recovery deploy; the up-front
check was one boot-log read (~100:1 asymmetry). The countersign could not have caught it (probes
brief-level premises at planning time; this premise didn't exist yet); the evaluator arrives
after the damage. Cleanest receipt: a same-session A/B — the identical premise checked against
telemetry for one bug (disproved, saved) and asserted unchecked for another (burned). Four
instances in one week on a premise-light project. Converges with the queue app's "reality over fixtures"
from the opposite direction: two projects independently produced the observation rule.

## The channel problem (meta-finding)

Three projects fed evidence up through three improvised channels: a field report (the game project), exit-
debrief companion docs (the queue app), a board line pointing at a finding (the booking app). None was a named
mechanism. v3 names one: local field notes, central harvest (see CONTRIBUTING.md).

## Coherence

Every v3 change is one law at a different seam: no claim outranks its evidence — a pass claims
only the tree it saw (stopping rule); a diagnosis claims nothing without an observation
(observation rule); a handoff claims completeness only after attacking itself (session end); a
branch claims value only by content-on-main (consolidation); an evaluation claims independence
only if its brief is auditable (committed briefs); a memory claims truth only against repo state
(memory hygiene).
